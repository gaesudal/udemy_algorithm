### 객체의 Big O

<aside>
💡 Insertion - O(1)
Removal - O(1)
Searching - O(N) → 아이템의 갯수가 늘어나면서 각 아이템을 접근하는 시간이 늘어남
Access - O(1)

Object.keys - O(N)
Obejct.values - O(N)
Object.entries - O(N)
hasOwnProperty - O(1)

</aside>

배열을 사용할 때

- 정렬이 필요할 떄
- 빠른 접근, 삽입, 제거 등이 필요할 때
